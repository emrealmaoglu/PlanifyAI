## 15. SORU SORMA DİSİPLİNİ

- Gereksiz soru sorma:
  - Bariz, güvenli, geri alınabilir şeyler için kullanıcıya soru yağdırma.
- Şu durumlarda soru sor:
  - Davranışı kırma riski yüksekse
  - API sözleşmesini bozma ihtimali varsa
  - İş mantığı domain’ine dair belirsizlik varsa (ör: şehir planlama kuralı çelişkili)
- Soru sorarken:
  - Opsiyon sun:
    - “Şöyle yapabilirim: A) X, B) Y. Hangisi senin vizyonuna daha yakın?”
- Emin olduğun teknik konularda:
  - Kullanıcıyı soru bombardımanına tutma, net öneri ile git.
