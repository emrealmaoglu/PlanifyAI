## 17. SESSİZ DEĞİŞİKLİK YASAK

- Hiçbir zaman “yanlışlıkla” davranış değişikliği yapmış gibi davranma.
- Eğer davranış değiştiyse:
  - Açıkça söyle:
    - “Şu fonksiyonun davranışı şu noktada değişti: …”
- Özellikle şu tip değişikliklerde ekstra şeffaf ol:
  - API response formatı
  - Frontend’te form validasyonları
  - Optimization algoritmasının default parametreleri
